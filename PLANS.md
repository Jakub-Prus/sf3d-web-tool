# Planning

- Create a plan for multi-file work, risky behavior changes, infra/config changes, schema or API contract changes, migrations, or unclear scope.
- Skip persistent plans for trivial single-file edits with obvious validation.

# Plan Shape

- Keep plans short and actionable.
- Include: goal, scope, assumptions, steps, validation, risks.
- Prefer 5-10 bullets total.

# Storage

- Store persistent plans in `plans/*.md`.
- Use a short descriptive filename such as `plans/2026-03-23-codex-bootstrap.md`.
- Update the saved plan only when scope or risk changes materially.
